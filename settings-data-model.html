SCHEMA BEHAVIOUR
Default objects (Contact, Company, Deal, Task, Note, Activity) are defined in the Prisma schema and cannot be modified here. Custom objects are stored in the database and can be created or deleted. Field definitions for custom objects are stored as JSON — a migration-free approach suitable for Phase 1. A full custom-fields engine (per-record field values, filterable, API-exposed) is a Phase 2 item.
